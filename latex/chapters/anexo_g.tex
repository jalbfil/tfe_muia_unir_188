\chapter{Capturas previstas del prototipo}
\label{cap:anexog}

\section*{G.1. Finalidad del anexo}

Este anexo deja preparada la documentacion visual del prototipo software v0.1.0. En la
version actual no se incorporan capturas de pantalla definitivas, porque estas deben
tomarse sobre una ejecucion estable del backend, la interfaz y Ollama local. El objetivo
es fijar que evidencias visuales deben anadirse, que debe mostrar cada una y como se
relacionan con los resultados descritos en los Capitulos~\ref{cap:8}, \ref{cap:9} y
\ref{cap:10}.

Las capturas no sustituyen a los artefactos reproducibles ni a las metricas. Su funcion
es facilitar la defensa del TFM mostrando que la arquitectura no se queda en una
descripcion abstracta, sino que existe una interfaz capaz de ejecutar el flujo completo:
entrada textual, extraccion de senales, priorizacion RuleFit-lite, explicacion local,
probabilidades P1--P4 y registro de feedback humano.

\section*{G.2. Criterios para incorporar capturas}

Las imagenes que se incorporen a este anexo deben cumplir los siguientes criterios:

\begin{itemize}
    \item proceder de una ejecucion local del prototipo v0.1.0;
    \item no contener datos personales, telefonos, direcciones completas ni informacion
    sensible de personas reales;
    \item usar incidentes sinteticos o casos anonimizados;
    \item mostrar claramente la prioridad recomendada, la distribucion de confianza y
    la explicacion generada;
    \item permitir verificar que Capa 2 decide la prioridad y que Capa 3 solo la
    explica;
    \item estar alineadas con el smoke test P1--P4 documentado en
    \texttt{artifacts/reports/prototype\_v0.1.0\_smoke\_test.md} y con la validacion
    reproducible Capa 1--Capa 2--Capa 3 documentada en
    \texttt{artifacts/reports/capa1\_capa2\_e2e\_v0.1.0.md}.
\end{itemize}

\section*{G.5. Relacion con la evidencia experimental}

Las capturas deben interpretarse como evidencia de usabilidad y ejecucion del prototipo,
no como evaluacion cuantitativa. La evaluacion principal sigue siendo la del
Capitulo~\ref{cap:9}: comparativa baseline experto, RuleFit-lite e \texttt{imodels},
recall P1, test temporal, control anti-\textit{leakage}, validacion interna y fidelidad
explicativa. La captura de variables de Capa 1 debe servir especialmente para conectar
la interfaz con la validacion \texttt{capa1\_capa2\_e2e\_v0.1.0}, donde el flujo
integrado supera 5/5 escenarios curados tras el saneamiento de negaciones.

Por tanto, este anexo queda preparado para completar la memoria con soporte visual una
vez se seleccionen las capturas definitivas de la interfaz.
