\chapter{Plantilla de validacion interna y casos revisables}
\label{cap:anexod}

\section*{D.1. Finalidad del anexo}

Este anexo documenta el instrumento de validacion interna utilizado para revisar
recomendaciones de prioridad P1--P4 sobre casos reales del Registro 112 Castilla y Leon.
Su funcion es cerrar el bucle humano del sistema: no basta con reportar metricas
automaticas, sino que los casos de riesgo y las discrepancias deben poder revisarse,
justificarse y registrarse de forma trazable.

La validacion descrita aqui no sustituye a una validacion externa con operadores del
112. Es una revision academica interna, adecuada para el alcance v0.1.0, que permite
detectar patrones de error, comprobar la comprensibilidad de las evidencias mostradas y
preparar una futura evaluacion operativa.

\section*{D.2. Muestra de revision}

La muestra se genera mediante \texttt{scripts/build\_internal\_validation\_sample.py}.
El artefacto principal es
\texttt{resources/internal\_validation/casos\_revision\_v0.1.0.csv}, acompa\~nado por
un resumen en \texttt{casos\_revision\_v0.1.0.md}. La seleccion combina dos criterios:

\begin{itemize}
    \item inclusion de los 51 falsos negativos P1 detectados en el test estratificado;
    \item inclusion de una muestra equilibrada adicional de 60 casos, con 15 casos por
    clase P1, P2, P3 y P4.
\end{itemize}

El total resultante es de 111 casos revisables. Esta cifra cubre el minimo de 30 casos
exigido para la validacion interna y deja preparada una base superior a 100 casos para
la evaluacion de fidelidad de explicaciones de T113, si se conecta posteriormente con
la Capa 3.

\section*{D.5. Procedimiento de revision}

Cada autor revisa de forma independiente los casos asignados, consultando el texto
operativo, la etiqueta debil, la recomendacion del sistema, las probabilidades, las
se\~nales activadas y, cuando este disponible, la explicacion normativa de Capa 3. La
decision se registra en tres valores posibles:

\begin{itemize}
    \item \textbf{Aceptar}: la recomendacion del sistema se considera adecuada.
    \item \textbf{Modificar}: el revisor asigna otra prioridad P1--P4 y justifica el
    cambio.
    \item \textbf{Rechazar}: el caso revela una recomendacion no util o evidencia
    insuficiente para sostenerla.
\end{itemize}

Las divergencias de dos o mas niveles se marcan como auditoria especial. En particular,
los casos P1 clasificados como P3 o P4 deben analizarse antes de congelar el modelo,
porque representan infraestimaciones relevantes. En la version evaluada no aparecen
casos P1--P4, pero si 19 casos P1--P3.

\section*{D.6. Limitaciones}

La muestra procede de etiquetas academicas generadas por supervision debil. Por tanto,
la revision interna puede detectar incoherencias del sistema o de la propia etiqueta,
pero no certifica validez operativa. La validacion externa con personal del 112 sigue
siendo una linea prioritaria de trabajo futuro.
