\chapter{Guia de etiquetado P1--P4}
\label{cap:anexoc}

\section*{C.1. Finalidad del anexo}

Este anexo documenta la guia usada para construir la etiqueta academica P1--P4 mediante
supervision debil. La guia no pretende reemplazar una prioridad oficial del 112, porque
el Registro 112 Castilla y Leon no contiene dicha etiqueta. Su funcion es definir un
criterio reproducible para entrenar, comparar y evaluar la Capa 2 dentro del alcance
academico del TFM.

La version operativa se corresponde con
\texttt{resources/labeling\_guide\_p1p4.md} y con la decision metodologica de tratar
P1--P4 como escala academica, no juridica ni institucional.

\section*{C.2. Principios generales}

\begin{enumerate}
    \item \textbf{Momento de primera decision.} La etiqueta debe reflejar la urgencia
    con la informacion disponible al inicio, no con el desenlace final del incidente.
    \item \textbf{Prioridad como urgencia relativa.} P1--P4 ordena necesidad de
    atencion temprana; no equivale a declaracion formal de situacion operativa.
    \item \textbf{Precaucion razonable.} Ante informacion incompleta, se evita tanto
    minimizar se\~nales criticas como asumir siempre el peor caso.
    \item \textbf{Trazabilidad.} Toda etiqueta debe poder vincularse a texto,
    categoria, senales, variables o reglas.
    \item \textbf{Independencia anti-\textit{leakage}.} No se usan medios movilizados,
    pacientes atendidos, cierre ni informacion posterior para etiquetar la primera
    decision.
\end{enumerate}

\section*{C.5. Resolucion de conflictos}

Cuando varias senales sugieren prioridades diferentes, se aplica la prioridad mas alta
compatible con la informacion inicial. La incertidumbre no debe rebajar una prioridad:
debe reflejarse como menor confianza o como caso para revision humana. Si la evidencia
solo permite una lectura ambigua entre dos niveles, se selecciona el nivel superior
cuando existan senales de personas afectadas, vulnerabilidad o posible escalado.

Los conflictos mas relevantes son:

\begin{itemize}
    \item \textbf{P1/P2}: se decide por la presencia de riesgo vital, fallecido,
    atrapamiento critico o gravedad personal extrema.
    \item \textbf{P2/P3}: se decide por potencial de escalado, personas expuestas,
    varias llamadas, incendio habitado o necesidad de rescate.
    \item \textbf{P3/P4}: se decide por existencia de gestion activa frente a consulta
    o incidencia menor.
\end{itemize}

\section*{C.6. Fuentes debiles y acuerdo}

La etiqueta final no procede de un unico criterio manual. Se construye combinando varias
fuentes debiles: reglas textuales, categoria operativa, senales de gravedad y criterios
de consistencia. El acuerdo entre fuentes se mide para verificar que la etiqueta no sea
puramente arbitraria. En la ejecucion v0.1.0, Krippendorff
\(\alpha = 0.858348\), y la ablacion sin reglas heuristicas mantiene
\(\alpha = 0.834155\).

Estos valores no convierten la etiqueta en prioridad oficial, pero si justifican su uso
como objetivo academico reproducible para entrenar y evaluar RuleFit-lite.

\section*{C.7. Limitaciones}

La guia reduce subjetividad, pero no elimina la necesidad de revision humana. Los 51
falsos negativos P1 detectados en test estratificado se incorporan a la muestra del
Anexo~\ref{cap:anexod} precisamente porque la etiqueta debil y la recomendacion del
modelo deben revisarse cuando el error potencial afecta a seguridad.

La validacion externa con operadores del 112 queda como trabajo futuro. Hasta entonces,
P1--P4 debe citarse siempre como escala academica del prototipo.
