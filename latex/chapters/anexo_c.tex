\section*{C.1. Finalidad del anexo}

% Este anexo desarrolla la guía de etiquetado del sistema de priorización, cuya versión resumida se presenta en el apartado~6.6 del Capítulo~6. La guía define cómo asignar una prioridad de referencia a cada caso del dataset piloto y cumple cuatro funciones complementarias: reducir la subjetividad, asegurar la consistencia interna, justificar la construcción del dataset y proporcionar el estándar de evaluación (\textit{ground truth}) para el Capítulo~9.

\section*{C.2. Principios generales de etiquetado}

% \paragraph{C.2.1. La prioridad es urgencia operativa relativa.}
% La prioridad mide la necesidad de atención temprana, no la importancia abstracta del suceso ni la gravedad final horas después.

% \paragraph{C.2.2. El etiquetado se realiza en el momento de la primera decisión.}
% La prioridad debe reflejar la urgencia con la información disponible en el instante inicial, no con el conocimiento posterior de la evolución.

% \paragraph{C.2.3. Precaución razonable, no dramatización.}
% Cuando la información sea incompleta, asumir un escenario moderadamente desfavorable, sin sistematizar el peor caso posible.

% \paragraph{C.2.4. Toda asignación debe ser justificable.}
% El etiquetador debe poder vincular su decisión a variables concretas, reglas duras o criterios operativos trazables.

% \paragraph{C.2.5. Independencia entre casos.}
% Cada caso se etiqueta de forma independiente. La distribución del dataset se logra mediante el diseño de los casos, no ajustando el etiquetado.

\section*{C.4. Reglas duras aplicables al etiquetado}

% Si un caso cumple la condición de una regla dura, la prioridad asignada no puede ser inferior a la establecida por esa regla, salvo justificación explícita y documentada. El procedimiento es: primero verificar reglas duras; si se activa una regla fija, la prioridad está determinada; si es mínima, la prioridad puede ser igual o superior al mínimo; si no se activa ninguna, la prioridad se determina por valoración de variables.

\section*{C.5. Criterios de resolución de conflictos}

% \paragraph{C.5.1. Prevalencia de la señal de mayor urgencia.}
% Cuando diferentes variables sugieren prioridades distintas, prevalece la señal de mayor urgencia compatible con la información disponible.

% \paragraph{C.5.2. Las reglas duras prevalecen sobre la valoración global.}
% Si una regla dura fija una prioridad mínima, ningún conjunto de variables favorables puede rebajar la prioridad por debajo de ese mínimo.

% \paragraph{C.5.3. La gravedad inmediata pesa más que el contexto.}
% En conflicto entre variables de gravedad inmediata (V01, V02, V03) y variables contextuales, prevalece la gravedad inmediata.

% \paragraph{C.5.4. La incertidumbre no rebaja la prioridad.}
% La incertidumbre se refleja en el nivel de confianza, no en la prioridad. Un incidente con posibles víctimas no confirmadas se etiqueta con la prioridad que correspondería al escenario plausible.

% \paragraph{C.5.5. El contexto solo eleva, no rebaja.}
% Las variables de amenaza contextual pueden elevar la prioridad cuando son pertinentes, pero su ausencia no rebaja una prioridad ya justificada por gravedad o vulnerabilidad.

% \paragraph{C.5.6. Documentación del conflicto.}
% Cuando se resuelva un conflicto, anotar brevemente qué variables entraron en conflicto y cuál fue el criterio de resolución.
