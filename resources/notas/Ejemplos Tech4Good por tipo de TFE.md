### Escenario Base: Mejorar el reciclaje mediante IA

#### 1. TFE Tipo 1: Piloto Experimental
**¿Qué se espera?** Medir el **impacto** de una tecnología en un entorno real con usuarios reales.
*   **Enfoque:** No se trata solo de crear el modelo, sino de implantarlo y ver si cambia el comportamiento de las personas.
*   **Ejemplo Tech4Good:** *"Impacto de un asistente visual basado en IA sobre la tasa de error en la clasificación de residuos en una cafetería universitaria."*
*   **Alcance del trabajo:**
    1.  Desplegáis una solución (puede ser una tablet sobre los cubos de basura que indique dónde tirar cada cosa usando visión artificial).
    2.  **Fase Real (Crucial):** Lo instaláis en la cafetería de la universidad durante 2 semanas.
    3.  **Análisis:** Comparáis los datos. ¿Se recicló mejor con la IA (Caso Experimental) que sin ella (Caso Base)?,.
*   **Reto principal:** Necesitáis acceso a un entorno físico, usuarios reales y permisos para medir resultados. Es un trabajo muy estadístico y de campo.

#### 2. TFE Tipo 2: Desarrollo Software
**¿Qué se espera?** Un **producto tecnológico funcional** y novedoso. El código y la herramienta son los protagonistas.
*   **Enfoque:** Crear una aplicación completa (Frontend + Backend + IA) que resuelva el problema, validando que sea fácil de usar.
*   **Ejemplo Tech4Good:** *"EcoScan: Desarrollo de una aplicación móvil gamificada para la identificación y clasificación de envases plásticos mediante Deep Learning."*
*   **Alcance del trabajo:**
    1.  Programar la App móvil (Android/iOS).
    2.  Entrenar e integrar el modelo de reconocimiento de imágenes en el móvil (Edge Computing o llamada a API).
    3.  Añadir funcionalidades de usuario: gamificación, mapa de puntos limpios, perfil de usuario.
    4.  **Validación:** Pruebas de usabilidad con usuarios (¿es fácil de usar? ¿falla la cámara?),.
*   **Reto principal:** La demo tiene que funcionar perfectamente. No vale solo con que el modelo de IA sea bueno; la App no puede fallar. Como sois dos, uno podría encargarse del modelo de IA y el backend, y el otro del desarrollo móvil y la experiencia de usuario (UX).

#### 3. TFE Tipo 3: Comparativa de Soluciones
**¿Qué se espera?** Investigación profunda y **empírica** (con datos). Buscar la mejor solución técnica para un problema específico.
*   **Enfoque:** No construís un producto para usuarios finales, sino que actuáis como científicos de datos buscando la arquitectura óptima.
*   **Ejemplo Tech4Good:** *"Estudio comparativo de arquitecturas YOLO vs EfficientDet para la detección de residuos peligrosos en plantas de triaje con baja iluminación."*
*   **Alcance del trabajo:**
    1.  Seleccionáis/Creáis un dataset complejo (ej. fotos de basura con mala luz, sucia, aplastada).
    2.  Entrenáis 3 o 4 modelos diferentes bajo las mismas condiciones rigurosas.
    3.  **Experimentación:** Medís métricas técnicas al detalle: *Accuracy*, *Recall*, tiempos de inferencia (ms), consumo de memoria GPU.
    4.  **Conclusión:** "El modelo X es un 15% más rápido, pero el modelo Y falla menos en condiciones oscuras",.
*   **Reto principal:** El rigor científico. No vale solo comparar teóricamente; tenéis que ejecutar experimentos controlados y justificar cada hiperparámetro.
